Important: This is NOT a Cryptocurrency Wallet

This extension is an authentication bridge that connects existing wallet extensions (like MetaMask) to the Crypto Trading Journal web application. It does NOT store funds, private keys, or seed phrases.

How It Works

(Existing Wallet)

(Auth Bridge)
→
Trading Journal
(Web App)

The extension facilitates Sign-In With Ethereum (SIWE) authentication. Users sign a message with their existing wallet, proving ownership without passwords.

What to Expect on cryptotradingjournal.xyz

When you visit the main website, you should see:

  1. Landing/Login Page - A clear "Connect Wallet" button
  2. MetaMask Popup - After clicking connect, MetaMask asks to approve connection
  3. SIWE Signature - A message to sign that includes the domain and your address
  4. Dashboard - After signing, you're logged in and see your dashboard

The "Connect Wallet" button should work without requiring you to create an account first. The wallet connection IS the authentication method.

If cryptotradingjournal.xyz is Unavailable

If the external website is temporarily unavailable, you can still verify the extension's functionality:

  1. Click extension icon - Status indicators prove extension loads correctly
  2. Check "Extension Active" - Should always show ✅
  3. Domain check - Shows ⚠️ with helpful guidance
  4. Review the code - Extension uses standard SIWE (EIP-4361) protocol

Note: The extension cannot demonstrate full auth flow without the backend, but the client-side code can be verified through status indicators.

Please contact us via GitHub Issues if you encounter any issues accessing the website during review.

What This Extension Does NOT Do

  • NOT a wallet - Cannot store, send, or receive cryptocurrency
  • NOT standalone - Requires both MetaMask AND the web app to function
  • NOT for key management - Never accesses or stores private keys
  • NOT for transactions - Only facilitates authentication signatures
